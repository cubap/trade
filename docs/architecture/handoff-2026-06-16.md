# Handoff Document — 2026-06-16

## Session Summary

Built a "through-the-head" 3D skull HUD for the first-person game view. The head mesh is a translucent purple ellipsoid with teddy bear ear nubs, positioned at the bottom of the screen so the crown drops off naturally. The head turns left/right based on pawn movement/studying, and the color shifts slightly to indicate turn direction.

---

## What Was Completed

### 3D Head Mesh (ThreeRenderer.js)
- **`_createHeadMesh()`**: Creates a merged geometry (head + ears as one mesh) with uniform translucency
  - Ellipsoid head: `SphereGeometry(1, 32, 24)` scaled 1.2×0.9×0.85
  - Two ear nubs: `SphereGeometry(0.22, 16, 12)` merged into the same geometry
  - Single `MeshStandardMaterial`: color `#8b7288`, opacity 0.35, DoubleSide, depthWrite false
  - Group scale: 2.5
- **`_mergeGeometries()`**: Manual BufferGeometry merge (no BufferGeometryUtils import needed)
- **`_updateHeadMesh()`**: Called each frame in `render()`
  - Positions head at bottom of screen using camera direction vectors
  - Distance from camera: 8 units, bottom offset: 4 units
  - Head faces camera via `lookAt()`
  - Yaw rotation from `this.headYaw` (set by thoughtDome): `head.rotation.y += headYaw * 0.5`
  - Color gradient shifts warm/cool based on turn direction
  - Health-based color shift (purple when healthy, reddish when unhealthy)

### Thought Dome (thoughtDome.js)
- **Head yaw calculation**: From pawn movement direction (`Math.atan2(dy, dx)`) or centered when studying
- **Smooth interpolation**: `turnSpeed = 0.06` (moving) or `0.02` (studying)
- **Camera bob**: Separate phase, ±2px, only when pawn is moving
- **Passes yaw to renderer**: `renderer.headYaw = headYaw`
- **Thought queue**: 3 stacked entries, 10s fade, 32px spacing

### App Integration (app.js)
- `thoughtDome = setupThoughtDome(() => trackedPlayerPawn, () => activeRenderer)`
- Renderer getter passes current renderer to thoughtDome
- ThreeRenderer constructor creates head mesh after camera (line 102)

---

## Design Decisions

1. **Merged geometry over separate meshes** — Uniform translucency, no overlapping transparency artifacts
2. **Scene child over camera child** — More control over positioning, camera-child approach had visibility issues
3. **Color gradient for turn direction** — Simpler than ear scale animation, works well with merged geometry
4. **Manual geometry merge** — Avoids importing `BufferGeometryUtils` (keeps bundle small, no bundler)

#### Over-the-Shoulder Camera (ThreeRenderer.js)
- Replaced first-person head-follow with over-the-shoulder follow camera
- **Camera tuning object** (defaults saved after user tuning):
  - `behindDistance: 1.7` — how far behind pawn
  - `cameraHeight: 3.0` — height above ground
  - `lookDistance: 9.0` — how far ahead to look
  - `lookHeight: 4.6` — height of look target
- **Pawn tuning**:
  - `pawnScale: 3.1`
  - `pawnYOffset: -0.1`
- **Smooth catch-up for yaw rotation**: `response: 0.18, damping: 0.88, maxSpeed: 0.12`

#### Camera Tuning UI (`solo/js/ui/cameraTuning.js`)

- Pause button (integrates with `controls.setPaused`)
- Reset button restores defaults
- Keyboard shortcuts: `C` to toggle panel, `Space` for pause
- Press `C` in-game to open

#### Smooth Pawn Movement (ThreeRenderer.js)
- **Problem**: Pawn jumped between discrete positions (prevX/prevY → x/y) at tick boundaries
- **Solution**: Two-layer interpolation:
  1. Progress-based: `interpX = prevX + (x - prevX) * progress` where progress is 0→1 within a tick
  2. Lerp smoothing: `_smoothedPawnPos` Vector3 with `0.35` lerp factor eliminates tick-boundary sawtooth
- `_updateHeadMesh(progress)`: Uses `_smoothedPawnPos` for model position
- `_updateCamera(progress)`: Uses interpolated positions for camera target
- `_smoothedPawnRotY`: Separate smoothed rotation with same lerp approach

#### Duplicate Pawn Fix (ThreeRenderer.js)
- **Problem**: Player pawn rendered twice — once as `_headMesh` (GLB), once as regular entity in the loop
- **Fix**: `render()` skips player pawn when `firstPersonLocked`: `if (e.id === this.playerPawnId && this.firstPersonLocked) continue`

#### Quiz Contextual Thoughts (`solo/js/ui/slowStartQuiz.js`)
- Each `SLOW_START_QUESTION` now has a `thought` property:
  - `firstImpulse`: "Where do I start?"
  - `resourcePriority`: "This could all be useful…"
  - `socialTendency`: "I'm not alone, am I?"
  - `longTermDrive`: "I guess this is home…"
- `showQuestion()` calls `pawn.addThought(question.thought, 'quiz')` when question fires
- Thought dome picks up these thoughts automatically

#### Journal Overlay (`solo/js/ui/journalOverlay.js`)
- New module: `setupJournal(pawnGetter)`
- Full-screen dark overlay with centered panel
- Shows all thoughtLog entries in reverse order (newest first)
- Tag-based color accents (quiz, spawn, hardship, insight, social, epiphany, loss)
- Each entry shows text, tick number, and tag
- Keyboard: `J` to toggle, `Escape` to close
- Close button (✕) in header
- Integrated in `app.js`: `const journal = setupJournal(() => trackedPlayerPawn)`

#### Thought Dome Fix (`solo/js/ui/thoughtDome.js`)
- **Problem**: Used `tick` as uniqueness check — multiple thoughts in same tick were dropped
- **Fix**: Track by log index (`lastThoughtIndex`) instead of tick value
- Now every new thought in the log gets displayed
