# Handoff Document — 2026-06-18

## Session Summary

Redesigned the river generation system in `TerrainGenerator.js` from a batch approach (simulate all 15 rivers, then post-process merges) to an iterative approach (pick sources one at a time, check convergence, stop when we have enough good rivers). Fixed a critical bug where the meandering perpendicular vector wasn't normalized, causing rivers to jump 300m laterally instead of 3m.

## What Was Completed

### River Generation Redesign (TerrainGenerator.js)

#### Problem History
1. **Rivers running parallel**: Multiple rivers followed the same global gradient without merging
2. **Rivers too straight**: 8-direction grid pathfinding created rigid lines
3. **Meandering bug**: Perpendicular vector wasn't normalized — offset was `meanderStrength × stepSize` (20 × 15 = 300m) instead of just `meanderStrength` (3m). Rivers zigzagged chaotically.
4. **Too many independent rivers**: Batch approach generated 15 rivers, parallel snapping was a weak post-process
5. **Tributary counts wrong**: Absorbed parallel rivers didn't transfer counts properly

#### Solution: Iterative Source Selection

**Old approach:**
- Simulate all 15 rivers upfront
- Post-process: parallel snap, tributary merge, prune, dedup
- Weak convergence — rivers rarely merged because they were all independent

**New approach (`generateRivers()`):**
1. Sample elevation grid → find high-elevation candidates (globalH > 60)
2. Sort by global height descending
3. Pick random sources from top candidates (up to `maxRivers × 3`)
4. **Loop**: Pick one source → simulate path → check if it joins an existing river → either boost tributaries or add as new river
5. **Convergence**: Stop when `rivers.length >= minRivers` AND `totalLength >= minTotalLength`

**Key parameters:**
```js
candidateSources = 80    // High points to sample
minPathLength = 10       // Minimum steps to survive
mergeProximity = 25      // Distance to count as joining (tight — real merges only)
minRivers = 5            // Stop once we have this many
minTotalLength = 300     // ...and total points across all rivers
maxRivers = 15           // Hard cap
maxJoinFraction = 0.3    // Only count as join if in first 30% of path
```

**Join logic:** A new path only "joins" an existing river if the connection happens early (first 30% of the new path, i.e., headwaters feeding into an existing channel). If it crosses downstream, it's treated as independent.

#### Meandering Fix
```js
// BEFORE (broken):
const perpX = -bestDy  // Not normalized!
const perpY = bestDx
// Offset = meanderStrength × stepSize = 20 × 15 = 300m

// AFTER (fixed):
const len = Math.sqrt(bestDx * bestDx + bestDy * bestDy)
const perpX = -bestDy / len  // Normalized
const perpY = bestDx / len
const meanderStrength = 3    // Meters of lateral offset
```

#### Path Simulation (`simulatePath()` helper)
- Extracted into a reusable function
- Uses `getGlobalHeight()` for descent direction (smooth gradient, no local minima)
- 8-direction search on `stepSize = 15` grid
- Lake drainage: when stuck, search 60m radius for escape, carve short channel (up to 5 attempts)
- Edge buffer: `edgeBuffer = 120` prevents rivers from reaching world border
- Meandering: 3m lateral offset per step (normalized perpendicular vector)

#### River Carving
Each river point has `{x, y, elev, tributaries}`. Carving scales with tributary count:
```js
carveWidth = riverBaseWidth + maxTributaries × riverWidthPerTributary  // 3 + n×4
carveDepth = riverCarvingDepth + maxTributaries × riverCarvePerTributary  // 15 + n×3
```

### Lake Drainage Channels
When a river gets stuck (no descent found), it searches a 60m radius for the lowest escape point. If found, it carves a short channel (depth = escapeDrop + 3, width = 6) and continues from the escape point. Up to 5 escape attempts before giving up.

## Design Decisions

- **Global height for pathfinding**: Use `getGlobalHeight()` (very smooth, 2-octave FBM at scale 1000) for river descent direction, not `getElevation()` (which has local detail noise causing local minima).
- **Iterative over batch**: Pick sources one at a time so later sources have a chance to join existing channels, creating natural tributary networks.
- **Convergence criteria**: Both a minimum river count AND minimum total length — prevents stopping too early with just 2-3 short rivers.
- **Join fraction limit**: Only count as a "join" if the connection is in the first 30% of the new path. This ensures the new path is truly a tributary feeding into an existing channel, not just crossing it.
- **Normalized meandering**: Perpendicular vector must be normalized, otherwise the offset scales with step size.
- **Edge buffer**: Rivers stop 120m from world edge (coastline), sources are 100m from edge.
